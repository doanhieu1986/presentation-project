Vibe Coding là gì?

Lập trình theo ý định — để AI lo phần còn lại.

"Vibe coding" là phong cách lập trình mà bạn mô tả ý định bằng ngôn ngữ tự nhiên, AI sinh ra code, và bạn chỉ cần review và iterate — không cần gõ từng dòng thủ công.

— Andrej Karpathy, tháng 2/2025

Tại sao nó thay đổi cách làm việc

🚀

Tốc độ prototype

Từ ý tưởng đến MVP trong vài giờ thay vì vài ngày. AI xử lý boilerplate, bạn tập trung vào logic nghiệp vụ.

🧠

Giảm cognitive load

Không cần nhớ syntax, API docs hay boilerplate patterns — AI làm thay. Bạn chỉ cần biết muốn gì.

🎯

Tập trung vào "What"

Chuyển tư duy từ "viết như thế nào" sang "cần đạt được gì" — đúng vai trò của engineer.

🔁

Iterate nhanh hơn

Thay đổi, refactor, thử nghiệm mà không sợ tốn thời gian. Feedback loop ngắn lại đáng kể.

Vibe Coding vs Traditional Coding

Khía cạnh Traditional Vibe Coding
Đầu vào Gõ code thủ công Mô tả bằng ngôn ngữ tự nhiên
Vai trò engineer Implementer Director + Reviewer
Tốc độ prototype Hours → Days Minutes → Hours
Yêu cầu kỹ năng Syntax mastery Problem framing + critical review
Rủi ro chính Human error, chậm AI hallucination — phải review kỹ

Quan trọng: Vibe Coding không thay thế kỹ năng engineering — nó khuếch đại nó. Engineer giỏi sẽ output tốt hơn nhiều lần. Engineer yếu sẽ tạo ra mess nhanh hơn nhiều lần.

Claude Code — Agentic Coding Assistant

CLI tool của Anthropic, chạy trực tiếp trong terminal và IDE.

Claude Code không chỉ trả lời câu hỏi — nó tự đọc codebase, tự chạy lệnh, tự sửa file, và hoàn thành task từ đầu đến cuối với sự giám sát tối thiểu từ phía bạn.

Khác gì so với AI chatbot thông thường

📁

Đọc toàn bộ codebase

Hiểu context của project, không chỉ đoạn code bạn paste vào chat.

⚙️

Chạy lệnh thực tế

Terminal, tests, build, install dependencies — tất cả tự động.

✏️

Sửa file trực tiếp

Edit, create, delete files — không cần copy-paste thủ công.

Cách hoạt động trong một task

  1. 1

    Bạn mô tả yêu cầu

    Bằng ngôn ngữ tự nhiên — ngắn gọn hoặc chi tiết tùy bạn và độ phức tạp của task.

  2. 2

    Claude đọc context

    Tự khám phá codebase, đọc CLAUDE.md, hiểu architecture và conventions của project.

  3. 3

    Lên kế hoạch (tùy chọn)

    Với task phức tạp, Claude đề xuất approach — bạn approve trước khi nó bắt đầu code.

  4. 4

    Thực thi

    Chạy lệnh, sửa file, tạo component — tất cả hiển thị minh bạch để bạn theo dõi.

  5. 5

    Bạn review & iterate

    Xem diff, chạy thử, cho feedback. Claude adjust nếu có vấn đề.

CLAUDE.md — Project Structure File

File context giúp Claude hiểu project của bạn ngay từ lần đầu tiên.

CLAUDE.md là file Markdown đặt ở root của project. Claude Code tự động đọc file này mỗi khi bắt đầu session — đây là cách bạn truyền context, conventions, và quy tắc của project cho AI một lần duy nhất.

Tại sao CLAUDE.md quan trọng

  • Nhất quán: Claude luôn code theo đúng style và conventions của team — không cần nhắc lại mỗi session.
  • Tiết kiệm thời gian: Không phải giải thích architecture hay tech stack mỗi lần mở conversation mới.
  • Giảm lỗi: Claude biết những gì cần tránh, những pattern nào đã deprecated trong project.
  • Ưu tiên cao nhất: Các hướng dẫn trong CLAUDE.md override behavior mặc định của Claude.
  • Onboarding kép: CLAUDE.md tốt cũng là tài liệu onboarding tốt cho developer mới vào team.

CLAUDE.md được đọc khi nào

✅ Tự động mỗi session

Claude đọc ngay khi bắt đầu — không cần bạn làm gì thêm hay nhắc đến file này.

✅ Hỗ trợ nhiều layer

Đặt ở root và thêm trong subfolder — Claude đọc tất cả theo context của task đang làm.

⚠️ Commit lên git

File này nên commit — là tài liệu chung của team. Tuyệt đối không chứa credentials hay secrets.

⚠️ Cần maintain

CLAUDE.md lỗi thời còn tệ hơn không có. Update ngay khi refactor lớn hoặc thay đổi convention.

Các file & thư mục quan trọng

Bản đồ đầy đủ của một Claude Code project — mỗi file có vai trò riêng.

Sơ đồ cấu trúc

Project Tree 
your-project/
├── README.md              ← Cho người đọc (human docs)
├── CLAUDE.md              ← Cho Claude (instructions, conventions)
├── SKILL.md               ← Domain knowledge tái sử dụng
│
├── .claude/               ← Thư mục cấu hình Claude Code
│   ├── settings.json      ← Permissions, model, hooks (commit lên git)
│   ├── settings.local.json ← Overrides cá nhân (gitignore)
│   ├── .mcp.json          ← MCP server integrations
│   ├── rules/             ← Instructions theo chủ đề, load on-demand
│   ├── skills/            ← Slash commands & reusable workflows
│   ├── agents/            ← Custom AI sub-agents chuyên biệt
│   └── hooks/             ← Shell scripts tự động hóa
│
├── templates/             ← Template files (domain-specific)
└── output/                ← Files được generate ra

README.md vs CLAUDE.md — Khác nhau như thế nào?

FileĐọc bởiNội dung
README.md Developer mới, stakeholder Tổng quan project, setup hướng dẫn, link tài liệu
CLAUDE.md Claude Code (AI) Conventions, commands, patterns, những gì Claude cần biết để code đúng

README giải thích "Project này làm gì."  CLAUDE.md giải thích "Claude phải làm theo cách nào."

SKILL.md & .claude/skills/ — Workflows tái sử dụng

Skills là các file hướng dẫn cho từng task cụ thể, được load on-demand thay vì luôn chiếm token context. Bạn gọi bằng /skill-name hoặc Claude tự nhận biết khi phù hợp.

🔧

SKILL.md ở root

Domain knowledge chung của project — Claude đọc khi làm task liên quan. Ví dụ: quy trình deploy, cách viết migration.

⚡

.claude/skills/

Mỗi skill là một thư mục với SKILL.md riêng. Hỗ trợ frontmatter để config model, tools, permissions cho từng workflow.

🔑

Ưu điểm

Tiết kiệm context — chỉ load khi cần. Dễ share trong team qua git. Có thể gán model riêng (haiku cho task đơn giản).

📋

Ví dụ dùng

/deploy, /review-pr, /commit, /test — mỗi cái là một skill file riêng.

.claude/settings.json — Trung tâm cấu hình

.claude/settings.json 
{
  "model": "claude-sonnet-4-6",
  "permissions": {
    "allow": ["Bash(npm *)", "Bash(git *)", "WebFetch"],
    "deny":  ["Bash(rm -rf *)", "Write(.env*)"]
  },
  "hooks": {
    "PostToolUse": [{
      "matcher": "Edit|Write",
      "hooks": [{ "type": "command", "command": "npx prettier --write" }]
    }]
  }
}

.claude/agents/ — Sub-agents chuyên biệt

Mỗi agent là một AI riêng với context window, tool access và permissions độc lập. Tạo file .claude/agents/code-reviewer.md:

.claude/agents/code-reviewer.md 
---
name: code-reviewer
description: Chuyên review code sau khi có thay đổi
tools: Read, Glob, Grep   # chỉ đọc, không sửa
model: sonnet
---

Bạn là senior engineer. Review code về:
- Correctness, edge cases
- Security vulnerabilities
- Performance & maintainability

.claude/hooks/ — Tự động hóa lifecycle

🔒

PreToolUse

Chặn lệnh nguy hiểm trước khi chạy. Ví dụ: block rm -rf ngoài /tmp.

✨

PostToolUse

Chạy sau khi tool xong. Ví dụ: auto-format file vừa edit bằng Prettier.

🔔

Notification

Gửi notification khi Claude cần input. Ví dụ: macOS notification popup.

Memory — Học và nhớ theo thời gian

Claude tự lưu learnings vào ~/.claude/projects/<project>/memory/ — không nằm trong repo. Gồm: build commands, debugging patterns, preferences của bạn. 200 dòng đầu của MEMORY.md được load tự động mỗi session.

MEMORY.md (index) user memories feedback memories project memories reference memories

Ví dụ thực tế — Next.js project

CLAUDE.md 
# My App — Claude Code Context

## Mục đích
E-commerce platform cho SME Việt Nam.
Stack: Next.js 14 (App Router), Supabase, Tailwind CSS, TypeScript.

## Cấu trúc
src/app/          ← App Router pages & layouts
src/components/   ← UI components (shadcn/ui base)
src/lib/          ← Utilities, hooks, API clients
src/types/        ← TypeScript types & interfaces
supabase/         ← Migrations & edge functions

## Conventions
- Component files: PascalCase (UserCard.tsx)
- Hooks: camelCase với prefix use (useCart.ts)
- Server components là default — dùng 'use client' chỉ khi cần
- CSS: Tailwind utility-first, không viết CSS module

## Commands
npm run dev       ← Dev server (port 3000)
npm run build     ← Production build
npm run test      ← Vitest unit tests

## Quan trọng — đọc kỹ trước khi code
- KHÔNG dùng useEffect để fetch data — dùng Server Components
- Tất cả API calls phải đi qua src/lib/api.ts
- Supabase RLS phải bật cho mọi table production
- Không thêm npm package mới nếu chưa hỏi

Những gì KHÔNG nên có

  • Credentials, API keys, secrets — file này commit lên git, ai cũng đọc được.
  • Quá dài (300+ dòng): Claude xử lý kém hiệu quả hơn, không phải hiệu quả hơn.
  • Thông tin lỗi thời: Refactor xong mà không update CLAUDE.md → AI sẽ code theo convention cũ.
  • Giải thích cái AI đã biết: Không cần giải thích React là gì hay cách dùng async/await.

Workflow Vibe Coding thực tế

Quy trình từ đầu đến cuối khi dùng Claude Code cho một feature mới.

Setup lần đầu (một lần duy nhất)

  1. 1

    Cài Claude Code

    Cài qua VS Code Extension hoặc CLI. Đăng nhập tài khoản Anthropic.

  2. 2

    Tạo CLAUDE.md ở root project

    Bắt đầu đơn giản: mục đích, tech stack, cấu trúc folder. Bổ sung dần theo nhu cầu thực tế.

  3. 3

    Commit CLAUDE.md vào git

    Share cho cả team — đây là tài liệu sống của project, không phải file cá nhân.

Làm một feature mới

  1. 1

    Mô tả feature bằng ngôn ngữ tự nhiên

    Càng cụ thể càng tốt — nêu rõ UI mong muốn, edge cases, constraint nếu có.

  2. 2

    Claude tự khám phá codebase

    Tìm component liên quan, hiểu data model, đề xuất approach phù hợp conventions trong CLAUDE.md.

  3. 3

    Review plan (với task phức tạp)

    Dùng Plan Mode: yêu cầu Claude đề xuất approach trước, confirm xong mới cho thực thi.

  4. 4

    Chạy và kiểm tra

    Test thủ công hoặc chạy test suite. Báo lỗi cụ thể cho Claude nếu có vấn đề.

  5. 5

    Iterate đến khi đúng

    Mô tả lỗi hoặc điều cần thay đổi — Claude fix và bạn review lại. Lặp đến khi xong.

Ví dụ prompt tốt vs không tốt

✅ Tốt — cụ thể, có context, có constraint

"Thêm validation email vào form đăng ký. Hiện error message inline dưới input, theo đúng style của form login hiện tại. Không dùng thư viện validation mới."

❌ Không tốt — quá mơ hồ

"Fix form đăng ký"

Best Practices & Tips

Những gì thực tế dùng Vibe Coding với Claude Code dạy lại.

Về CLAUDE.md

  • Update ngay sau refactor lớn: CLAUDE.md lỗi thời dẫn đến AI code theo pattern cũ — tệ hơn không có file.
  • Layer theo folder: Đặt CLAUDE.md con trong thư mục phức tạp (vd: src/payments/CLAUDE.md) cho context cụ thể hơn.
  • Ngắn gọn wins: File 100 dòng được viết tốt hiệu quả hơn file 400 dòng đầy đủ nhưng loãng.
  • Viết "ghi chú quan trọng" trước: Những gotchas và anti-patterns thường quan trọng hơn mô tả tech stack.

Về Prompting

  • Cho ví dụ cụ thể: "Viết theo style như UserCard.tsx" tốt hơn mô tả trừu tượng.
  • Chia nhỏ task lớn: Thay vì "build toàn bộ checkout flow", chia thành các bước nhỏ, có thể verify từng bước.
  • Dùng Plan Mode: Với task phức tạp hoặc rủi ro cao, yêu cầu Claude lên kế hoạch trước khi code.
  • Chỉ rõ constraint: "Không thêm dependency mới", "Giữ nguyên API contract" — Claude sẽ tuân thủ.

Về Review & Safety

  • Luôn đọc diff: Vibe coding ≠ blind trust. Review mọi thay đổi trước khi commit, đặc biệt logic business.
  • Chạy tests: AI có thể pass một chỗ nhưng break chỗ khác — chạy test suite sau mỗi thay đổi lớn.
  • Commit nhỏ và thường xuyên: Dễ rollback khi cần. git commit sau mỗi feature nhỏ hoạt động được.
  • Không merge code bạn không hiểu: Nếu không hiểu logic AI sinh ra, đừng đưa vào production trước khi review kỹ.

Bottom line: CLAUDE.md tốt + prompt rõ ràng + review nghiêm túc = Vibe Coding hiệu quả. Bỏ bất kỳ một trong ba yếu tố này, kết quả sẽ không đáng tin cậy.
